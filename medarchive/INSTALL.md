# MedArchive — установка обновлённого фронта

В архиве только папка `src/` (полностью переписанная). Конфиги, package.json
и node_modules не трогаются — они у тебя уже есть.

## Шаги

1. Закрой dev server (Ctrl+C в терминале).

2. В корне проекта `medarchive` удали старую папку src и распакуй новую:

   ```powershell
   Remove-Item -Recurse -Force src
   Expand-Archive -Path путь\к\medarchive-src.zip -DestinationPath .
   ```

   (Expand-Archive создаст medarchive/src — если архив распаковался в
   подпапку `medarchive`, перенеси из неё `src` в корень проекта.)

3. Поставь недостающие зависимости (новых нет, но на всякий случай):

   ```powershell
   npm install
   ```

4. Очисти кэш Vite и запусти:

   ```powershell
   Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
   npm run dev
   ```

5. Открой http://localhost:5173 в новой вкладке.

## Что есть

8 страниц с живыми моками (единый seed — данные консистентны между страницами):

- Дашборд — KPI, график нормализации, статусы документов, топ-партнёры
- Поиск услуг — услуга → клиники с ценами резидент/нерезидент
- Партнёры — фильтры + сетка карточек
- Карточка партнёра — вкладки: документы / услуги / история цен
- Справочник услуг — таблица с категориями и синонимами
- Очередь верификации — split-view ревью (ядро UX-критерия)
- Загрузка архива — drag&drop + симуляция пайплайна обработки
- Настройки — порог автосопоставления, курсы валют, версионирование

## Подключение реального бэкенда

Все страницы ходят только в `src/services/*`. Сейчас сервисы читают
`src/api/mock/dataset.ts`. Когда поднимешь FastAPI — в `src/services/http.ts`
поставь `USE_MOCK_API = false` и замени тела функций в сервисах на вызовы
`http.get(...)`. Компоненты менять не нужно.
